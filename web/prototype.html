What you're looking at

A small SPI NOR region running a log-structured filesystem (the LittleFS / SPIFFS / FASTFFS idea). NOR has one governing asymmetry:

  • Program clears bits 1→0 at byte granularity — and you can keep programming into 0xFF space without erasing, because writing 0xFF clears nothing.
  • To set bits back to 1 you must erase, and erase only works on a whole 4 KB sector — the slow, wearing operation.

Each cell is a 256 B page; its fill height is the bytes programmed. The program granule rounds every write up and pads with 0xFF — raise it and watch padding waste climb. Append grows a file into its last page's 0xFF slack in place, no erase — the move NAND can't make.

Updates never overwrite: they append a fresh copy and mark the old one stale. When free pages run low the garbage collector takes the sector with the most stale pages, relocates survivors, and erases it — bumping that sector's wear. Flip on the heatmap to see wear pool into hotspots (greedy GC, no wear leveling yet).

Endurance is scaled to 200 cycles so wear-out is watchable; real NOR sectors survive ~100k. Space toggles run/pause.
